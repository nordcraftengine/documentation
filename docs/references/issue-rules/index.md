# Issue rules

Nordcraft's issue rules help you maintain high-quality standards in your projects by highlighting common problems.

::: tip
To learn more about where issue rules are shown in the editor, check out the [issues panel](/the-editor/issues-panel).
:::

::: tip
To see how issue rules are implemented, visit the [Nordcraft GitHub repository](https://github.com/nordcraftengine/nordcraft/tree/main/packages/search/src/rules).
:::

## Components

| Rule                                                                                                                                                                | Severity |      Auto fixes      | Description                                                                                                                            |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | :------------------: | -------------------------------------------------------------------------------------------------------------------------------------- |
| **[Components with no references](https://github.com/nordcraftengine/nordcraft/blob/main/packages/search/src/rules/issues/components/noReferenceComponentRule.ts)** | Warning  | Delete the component | Components that are not referenced anywhere in your project are usually safe to remove unless they're used as exported web components. |
| **[Unknown component reference](https://github.com/nordcraftengine/nordcraft/blob/main/packages/search/src/rules/issues/components/unknownComponentRule.ts)**       | Error    |          No          | Finds references to components that do not exist. This helps you identify and correct broken references in your project.               |
